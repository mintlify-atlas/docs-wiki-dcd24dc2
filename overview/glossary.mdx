This page is the terminology reference for `abpframework/abp`. Every term defined here appears repeatedly across the codebase — in module names, attributes, interfaces and metadata files — and each is grounded in a concrete file or symbol an agent can open. Use it as a lookup index when reading other wiki pages: every other overview/flow/module page assumes these meanings.

## How to read this glossary

<Info>
Each table row lists: **term** (canonical wording), **definition** (one paragraph, ABP-specific not generic DDD), **defining file or symbol** (clickable when local), **related wiki page** (deeper dive). When a term has parallel "abstractions" and "implementation" packages, both are listed.
</Info>

## Module system

| Term | Definition | Defining file / symbol | Related |
| --- | --- | --- | --- |
| **ABP Module** | A unit of composition. A class deriving from `AbpModule` that registers services in `ConfigureServices` and runs init code in `OnApplicationInitialization`. NuGet packages typically ship one module class each. | `framework/src/Volo.Abp.Core/Volo/Abp/Modularity/IAbpModule.cs`, `AbpModule.cs` | `/overview/architecture`, `/core/overview` |
| **Module Class** | Concrete class name ending in `Module` (e.g. `AbpIdentityDomainModule`). Marked with `[DependsOn(typeof(OtherModule), …)]`. | `AbpModule.IsAbpModule(Type)` in `AbpModule.cs` | `/overview/architecture` |
| **[DependsOn]** | Attribute declaring static dependencies between modules. Read by `IModuleLoader` to topologically order initialization. | `framework/src/Volo.Abp.Core/Volo/Abp/Modularity/DependsOnAttribute.cs` | `/overview/architecture` |
| **Startup Module** | The single root module type passed to `AbpApplicationFactory.CreateAsync<T>` / `builder.AddApplicationAsync<T>`. Pulls in every other module transitively. | `AbpApplicationBase.StartupModuleType` in `AbpApplicationBase.cs` | `/flows/application-startup` |
| **AbpApplication** | The runtime object owning `Services`, `ServiceProvider`, `Modules`, `InstanceId`, lifecycle methods. Abstract; created by `AbpApplicationFactory`. | `framework/src/Volo.Abp.Core/Volo/Abp/AbpApplicationBase.cs`, `IAbpApplication.cs` | `/overview/architecture` |
| **IModuleLoader** | Walks the `[DependsOn]` graph and emits a topologically ordered `IReadOnlyList<IAbpModuleDescriptor>`. | `framework/src/Volo.Abp.Core/Volo/Abp/Modularity/IModuleLoader.cs`, `ModuleLoader.cs` | `/flows/application-startup` |
| **IModuleManager** | Runs lifecycle contributors over the descriptor list (`Pre`/`Configure`/`Post` services, `Pre`/`On`/`Post` initialization, shutdown). | `framework/src/Volo.Abp.Core/Volo/Abp/Modularity/IModuleManager.cs`, `ModuleManager.cs` | `/flows/application-startup` |
| **ServiceConfigurationContext** | Passed to `AbpModule.ConfigureServices`. Exposes `Services` (`IServiceCollection`) and `Items` for cross-module hand-off. | `framework/src/Volo.Abp.Core/Volo/Abp/Modularity/ServiceConfigurationContext.cs` | `/core/overview` |
| **ApplicationInitializationContext** | Passed to `OnApplicationInitialization`. Holds the root `IServiceProvider` and a typed `GetEnvironment()`-style helper. | `framework/src/Volo.Abp.Core/Volo/Abp/Modularity/ApplicationInitializationContext.cs` | `/flows/application-startup` |
| **PlugIn source** | Optional dynamic module assembly (folder, type-list) added via `options.PlugInSources` in `AbpApplicationCreationOptions`. | `framework/src/Volo.Abp.Core/Volo/Abp/Modularity/PlugIns/` | `/core/overview` |

## DDD layers

| Term | Definition | Defining file / symbol | Related |
| --- | --- | --- | --- |
| **Domain.Shared** | Layer of constants, enums, error codes and shared localization. Referenced by **every other layer**. Has zero dependencies on persistence. | Per-module `Volo.Abp.<X>.Domain.Shared` csproj. Framework base: `Volo.Abp.Ddd.Domain.Shared` | `/ddd/overview` |
| **Domain** | Aggregate roots, entities, value objects, domain services, repository **interfaces**. Depends on `Domain.Shared`. | Framework base: `framework/src/Volo.Abp.Ddd.Domain/` | `/ddd/overview` |
| **Application.Contracts** | Application service **interfaces**, DTOs, permission/feature/setting definitions. Shipped to clients (e.g. Blazor WebAssembly) along with `HttpApi.Client`. | Framework base: `framework/src/Volo.Abp.Ddd.Application.Contracts/` | `/ddd/overview` |
| **Application** | Application service **implementations**. Coordinates domain services and infrastructure. Exposed to UI through `HttpApi` controllers. | Framework base: `framework/src/Volo.Abp.Ddd.Application/Volo/Abp/Application/Services/ApplicationService.cs` | `/ddd/overview` |
| **HttpApi** | ASP.NET Core controllers translating Application services into REST endpoints. | Module pattern: `modules/identity/src/Volo.Abp.Identity.HttpApi/` | `/ddd/overview` |
| **HttpApi.Client** | Dynamic C# proxies for remote consumption, with embedded `*generate-proxy.json` swagger. | `common.props` block `Condition="$(AssemblyName.EndsWith('HttpApi.Client'))"` | `/modules/overview` |
| **DDD Layer** | Generic label for any of the seven canonical layers above. Encoded in `.abppkg` `role` field (e.g. `lib.domain`). | `modules/identity/src/Volo.Abp.Identity.Domain/Volo.Abp.Identity.Domain.abppkg` = `{ "role": "lib.domain" }` | `/overview/solution-structure` |
| **Aggregate Root** | Cluster entity with a global identity that is the consistency boundary. Inherit from `AggregateRoot<TKey>` (with auditing) or `BasicAggregateRoot<TKey>` (no auditing). | `framework/src/Volo.Abp.Ddd.Domain/Volo/Abp/Domain/Entities/AggregateRoot.cs`, `BasicAggregateRoot.cs` | `/ddd/overview` |
| **Entity** | Has identity but is not an aggregate root. Inherit from `Entity<TKey>`. | `framework/src/Volo.Abp.Ddd.Domain/Volo/Abp/Domain/Entities/Entity.cs`, `IEntity.cs` | `/ddd/overview` |
| **Value Object** | Identity-less object compared by value. Inherit from `ValueObject` (records equality logic). | `framework/src/Volo.Abp.Ddd.Domain/Volo/Abp/Domain/Values/ValueObject.cs` | `/ddd/overview` |
| **Domain Event** | Event raised by an aggregate (`AddLocalEvent`/`AddDistributedEvent`). Dispatched on SaveChanges via the event bus. | `framework/src/Volo.Abp.Ddd.Domain/Volo/Abp/Domain/Entities/Events/`, `IGeneratesDomainEvents.cs` | `/eventbus/overview` |

| **Unit of Work** | Ambient transaction-and-change-tracking scope. Begins automatically around controller actions / app service calls. | `framework/src/Volo.Abp.Uow/Volo/Abp/Uow/IUnitOfWork.cs`, `IUnitOfWorkManager.cs`, `AmbientUnitOfWork.cs` | `/data/overview` |

| **IDatabaseApi / ITransactionApi** | Per-database handles attached to the UoW: EF's `DbContext` is an `IDatabaseApi`, the EF transaction is an `ITransactionApi`. | `framework/src/Volo.Abp.Uow/Volo/Abp/Uow/IDatabaseApi.cs`, `ITransactionApi.cs` | `/data/overview` |
| **Specification** | Encapsulated predicate over entities. Combine with `And`/`Or`/`Not`. Used as `IRepository.GetListAsync(ISpecification<T>)`. | `framework/src/Volo.Abp.Specifications/Volo/Abp/Specifications/ISpecification.cs`, `Specification.cs` | `/ddd/overview` |

## Cross-cutting infra

| Term | Definition | Defining file / symbol | Related |
| --- | --- | --- | --- |
| **Auditing** | Automatic capture of who/when changed which entity. Implement `IAuditedObject` or just inherit `AggregateRoot<TKey>`. `[Audited]` attribute records request-level audit logs. | `framework/src/Volo.Abp.Auditing/`, `IAuditingManager.cs`, `AuditedAttribute.cs`, `IAuditedObject.cs` | `/core/overview` |
| **Permission** | Authorization unit declared by `PermissionDefinitionProvider`, checked via `IPermissionChecker.IsGrantedAsync("MyPerm")` or `[Authorize("MyPerm")]`. | `framework/src/Volo.Abp.Authorization/Volo/Abp/Authorization/Permissions/IPermissionChecker.cs`, `PermissionDefinitionProvider.cs` | `/core/overview` |
| **Feature** | Tenant-scoped flag/value defined by `FeatureDefinitionProvider`. Read via `IFeatureChecker.IsEnabledAsync("MyFeature")` or `[RequiresFeature("MyFeature")]`. | `framework/src/Volo.Abp.Features/Volo/Abp/Features/IFeatureChecker.cs`, `FeatureDefinitionProvider.cs` | `/core/overview` |
| **Setting** | Named string value with override hierarchy (default → global → tenant → user). Defined by `SettingDefinitionProvider`, read via `ISettingProvider.GetOrNullAsync("My.Setting")`. | `framework/src/Volo.Abp.Settings/Volo/Abp/Settings/ISettingProvider.cs`, `SettingDefinitionProvider.cs` | `/core/overview` |
| **Current Tenant** | Ambient `ICurrentTenant` exposing `Id`, `Name`, `IsAvailable`. `Change(tenantId)` opens an `IDisposable` scope. | `framework/src/Volo.Abp.MultiTenancy/Volo/Abp/MultiTenancy/CurrentTenant.cs`, `AsyncLocalCurrentTenantAccessor.cs` | `/multitenancy/overview` |
| **Tenant Resolve Contributor** | Pluggable step that inspects the request (header, claim, host, querystring, cookie, route) to determine `TenantId`. Chained until one yields. | `framework/src/Volo.Abp.MultiTenancy/Volo/Abp/MultiTenancy/TenantResolver.cs`, `TenantResolveContributorBase.cs`, `CurrentUserTenantResolveContributor.cs`, `ActionTenantResolveContributor.cs` | `/multitenancy/overview` |
| **Multi-Tenant Connection String** | `IConnectionStringResolver` that picks per-tenant database. ABP ships `MultiTenantConnectionStringResolver`. | `framework/src/Volo.Abp.MultiTenancy/Volo/Abp/MultiTenancy/MultiTenantConnectionStringResolver.cs` | `/multitenancy/overview` |
| **Data Filter** | Toggle-able global query filter (`ISoftDelete`, `IMultiTenant`). Enable/disable via `IDataFilter.Enable<T>()` / `Disable<T>()` returning `IDisposable`. | `framework/src/Volo.Abp.Data/Volo/Abp/Data/IDataFilter.cs`, `DataFilter.cs` | `/data/overview` |
| **MultiLingual Object** | Pattern for entities with translatable fields: a translation collection + helper `IMultiLingualObjectManager`. | `framework/src/Volo.Abp.MultiLingualObjects/` | `/data/overview` |
| **IObjectMapper** | Abstraction over object-to-object mappers. AutoMapper or Mapperly profiles register against it. | `framework/src/Volo.Abp.ObjectMapping/Volo/Abp/ObjectMapping/IObjectMapper.cs`, `Volo.Abp.AutoMapper/`, `Volo.Abp.Mapperly/` | `/core/overview` |
| **Object Extending** | Runtime extra-property bag (`HasExtraProperties.ExtraProperties` + `IObjectExtensionManager`) — extend entities without schema changes. | `framework/src/Volo.Abp.ObjectExtending/` | `/ddd/overview` |

| Term | Definition | Defining file / symbol | Related |
| --- | --- | --- | --- |
| **Local Event Bus** | In-process `IEventBus` dispatching to `ILocalEventHandler<TEvent>`. Fires synchronously inside the UoW. | `framework/src/Volo.Abp.EventBus/Volo/Abp/EventBus/Local/ILocalEventBus.cs` | `/eventbus/overview` |
| **Distributed Event Bus** | Process-boundary-crossing `IDistributedEventBus`. Implementations: RabbitMQ, Kafka, Azure Service Bus, Dapr, Rebus. | `framework/src/Volo.Abp.EventBus/Volo/Abp/EventBus/Distributed/IDistributedEventBus.cs` + `Volo.Abp.EventBus.RabbitMQ`, `.Kafka`, `.Azure`, `.Dapr`, `.Rebus` | `/eventbus/overview` |
| **Outbox / Inbox** | Transactional event delivery: published events first written to an `Outbox` table inside the same transaction; `OutboxSender` ships them later. Consumers dedupe via `Inbox`. | `framework/src/Volo.Abp.EventBus/Volo/Abp/EventBus/Distributed/OutboxSender.cs`, `OutboxSenderManager.cs`, `InboxProcessor.cs`, `InboxProcessManager.cs`, `InboxProcessorFailurePolicy.cs` | `/eventbus/overview` |
| **Background Job** | Durable, replayable work unit. Submit via `IBackgroundJobManager.EnqueueAsync<TArgs>(args)`. Handler implements `IBackgroundJob<TArgs>`. | `framework/src/Volo.Abp.BackgroundJobs.Abstractions/Volo/Abp/BackgroundJobs/IBackgroundJobManager.cs` | `/backgroundjobs/overview` |
| **Background Worker** | Long-running hosted service (`IBackgroundWorker`). ABP integrations: Hangfire, Quartz, TickerQ. | `framework/src/Volo.Abp.BackgroundWorkers/Volo/Abp/BackgroundWorkers/IBackgroundWorker.cs` | `/backgroundjobs/overview` |
| **Distributed Lock** | `IAbpDistributedLock.TryAcquireAsync` for cross-process mutual exclusion. Defaults to local; production uses Redis (`DistributedLock.Redis`). | `framework/src/Volo.Abp.DistributedLocking.Abstractions/Volo/Abp/DistributedLocking/IAbpDistributedLock.cs` | `/core/overview` |

## Naming conventions used everywhere

| Convention | Where it appears | Example |
| --- | --- | --- |
| `Volo.Abp.*` | Every framework / first-party module NuGet | `Volo.Abp.Core`, `Volo.Abp.Identity.Domain` |
| `Abp*Module` | Public framework module class name | `AbpAutofacModule`, `AbpDddDomainModule`, `AbpAspNetCoreMvcModule` |
| `Abp*Options` | MS Options pattern object for an ABP feature | `AbpMultiTenancyOptions`, `AbpLocalizationOptions`, `AbpVirtualFileSystemOptions`, `AbpBlobStoringOptions` |
| `Abp*ConventionalRegistrar` | Class scanning assemblies during `ConfigureServices` | `AbpRepositoryConventionalRegistrar`, `BasicConventionalRegistrar` |
| `Default*` | Default DI implementation of an `I*` abstraction | `DefaultModuleLifecycleContributor`, `DefaultConnectionStringResolver` |
| `Null*` | No-op fallback registered when no real implementation is configured | `NullTenantResolveResultAccessor`, `NullEmailSender` |
| `*Contributor` | Pluggable participant in a pipeline (resolve / register / bundle) | `TenantResolveContributorBase`, `IBundleContributor`, `ISettingValueProvider` |
| `*Provider` | Strategy supplying a value or feature implementation | `IBlobProvider`, `IFeatureValueProvider`, `IVirtualFileProvider` |
| `*Manager` | Domain-service-style coordinator | `IUnitOfWorkManager`, `IBlobContainerFactory`, `IAuditingManager` |
| `*Store` | Persistence-only abstraction sitting under a Manager | `IPermissionStore`, `IFeatureStore`, `ISettingStore` |
| `*Definition` / `*DefinitionContext` | Metadata records emitted by definition providers | `PermissionDefinition`, `FeatureDefinitionContext`, `SettingDefinition` |
| `Current*` | Ambient request-scoped accessor | `ICurrentUser`, `ICurrentTenant`, `ICurrentPrincipalAccessor` |
| `ITransient/Scoped/SingletonDependency` | Lifetime markers — convention picks them up | see Lifecycle section |
| `[Dependency]`, `[ExposeServices]`, `[UnitOfWork]`, `[DependsOn]`, `[Audited]`, `[DisableAuditing]`, `[Authorize(...)]`, `[RequiresFeature(...)]` | Attributes layered on classes/methods | All in `framework/src/Volo.Abp.*` |

## File / folder conventions inside a layer

| File or folder | Layer | Purpose |
| --- | --- | --- |
| `<Name>Module.cs` | every layer | The module class. One per csproj. |
| `<Name>Consts.cs` | Domain.Shared | Const strings (table prefixes, db schema, max lengths). Example `IdentityConsts`. |
| `<Name>ErrorCodes.cs` | Domain.Shared | Localized error-code constants thrown via `BusinessException`. |
| `Localization/Resources/<Name>/<lang>.json` | Domain.Shared | Localization JSON consumed via the VFS. |
| `<Aggregate>.cs` | Domain | Aggregate root POCO. |
| `<Aggregate>Manager.cs` | Domain | Domain service operating over the aggregate. |
| `I<Aggregate>Repository.cs` | Domain | Repository interface; implementation lives in EF/Mongo layer. |
| `<Entity>EventHandler.cs` | Domain | Local event handler for domain events. |
| `Permissions/<Name>PermissionDefinitionProvider.cs` | Application.Contracts | Permission tree definitions. |
| `Features/<Name>FeatureDefinitionProvider.cs` | Application.Contracts | Feature tree definitions. |
| `Settings/<Name>SettingDefinitionProvider.cs` | Domain | Setting tree definitions. |
| `I<Aggregate>AppService.cs` + `<Aggregate>Dto.cs` + `CreateUpdate<Aggregate>Dto.cs` + `<Aggregate>GetListInput.cs` | Application.Contracts | App service surface. |
| `<Aggregate>AppService.cs` + `<Name>ApplicationAutoMapperProfile.cs` | Application | Implementation + mapping. |
| `<Aggregate>Controller.cs` (RemoteService route) | HttpApi | Optional explicit controller; otherwise application services are auto-exposed. |
| `<Name>DbContext.cs` + `<Aggregate>DbContextModelCreatingExtensions.cs` + `EfCore<Aggregate>Repository.cs` | EntityFrameworkCore | EF persistence. |
| `<Name>MongoDbContext.cs` + `MongoDb<Aggregate>Repository.cs` | MongoDB | Mongo persistence. |
| `Pages/<Aggregate>/`, `Menus/<Name>MenuContributor.cs`, `Views/`, `Components/` | Web | Razor UI surface. |

<Tip>
The conventions above are not enforced by analyzers — they are enforced by ABP Studio's templates and the existing source. Putting a Permission Definition Provider in `Domain` instead of `Application.Contracts` will compile but will not be visible to clients that only reference the contracts. Always copy the layout of an existing module like `modules/identity/src/` when adding a new one.
</Tip>
