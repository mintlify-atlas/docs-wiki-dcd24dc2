The Aliyun provider package is `framework/src/Volo.Abp.BlobStoring.Aliyun/Volo/Abp/BlobStoring/Aliyun/`. It uses the **`Aliyun.OSS.SDK`** for the data plane and **`aliyun-net-sdk-sts`** for the optional Security Token Service path. Like the AWS provider, it depends on `AbpCachingModule` so STS sessions can be cached encrypted in `IDistributedCache<AliyunTemporaryCredentialsCacheItem>`.

## `UseAliyun(...)`

```csharp
containerConfiguration.ProviderType = typeof(AliyunBlobProvider);
containerConfiguration.NamingNormalizers.TryAdd<AliyunBlobNamingNormalizer>();
aliyunConfigureAction(new AliyunBlobProviderConfiguration(containerConfiguration));
```

Direct sub-account configuration:

```csharp
o.Containers.ConfigureDefault(c =>
{
    c.UseAliyun(a =>
    {
        a.AccessKeyId     = config["Aliyun:AccessKeyId"];
        a.AccessKeySecret = config["Aliyun:AccessKeySecret"];
        a.Endpoint        = "oss-cn-hangzhou.aliyuncs.com";
        a.ContainerName   = "myapp-blobs";
        a.CreateContainerIfNotExists = true;
    });
});
```

STS / AssumeRole:

```csharp
a.UseSecurityTokenService = true;
a.AccessKeyId     = config["Aliyun:RamUser:AccessKeyId"];     // RAM user that can AssumeRole
a.AccessKeySecret = config["Aliyun:RamUser:AccessKeySecret"];
a.RegionId        = "cn-hangzhou";
a.RoleArn         = "acs:ram::1234567890:role/oss-blob-role";
a.RoleSessionName = "blob-storing";
a.DurationSeconds = 3600;   // 900..3600
a.Policy          = "{ \"Statement\": [...] }";
```

## `AliyunBlobProviderConfiguration`

| Property | Bag key | Type | Default | Notes |
| --- | --- | --- | --- | --- |
| `AccessKeyId` | `Aliyun.AccessKeyId` | `string` | required | RAM/sub-account access key id. |
| `AccessKeySecret` | `Aliyun.AccessKeySecret` | `string` | required | Paired secret. |
| `Endpoint` | `Aliyun.Endpoint` | `string` | required | Region endpoint hostname (e.g. `oss-cn-hangzhou.aliyuncs.com`). |
| `UseSecurityTokenService` | `Aliyun.UseSecurityTokenService` | `bool` | `false` | Switch to STS-issued temporary credentials. When true, the four `Role*` and `DurationSeconds`/`Policy` properties become relevant. |
| `RegionId` | `Aliyun.RegionId` | `string` | required if STS | STS region id (`cn-hangzhou` etc.). |
| `RoleArn` | `Aliyun.RoleArn` | `string` | required if STS | `acs:ram::<accountID>:role/<roleName>`. |
| `RoleSessionName` | `Aliyun.RoleSessionName` | `string` | required if STS | Identifier for the temporary credentials. |
| `DurationSeconds` | `Aliyun.DurationSeconds` | `int` | `0` | 900-3600 seconds. |
| `Policy` | `Aliyun.Policy` | `string` | required if STS | Session policy JSON; an empty policy grants the role's full permissions. |
| `ContainerName` | `Aliyun.ContainerName` | `string?` | `null` | Override the bucket name; falls back to `args.ContainerName`. Run through `BlobNormalizeNamingService.NormalizeContainerName`. |
| `CreateContainerIfNotExists` | `Aliyun.CreateContainerIfNotExists` | `bool` | `false` | Create the bucket on first save if missing. |
| `TemporaryCredentialsCacheKey` | `Aliyun.TemporaryCredentialsCacheKey` | `string?` | `Guid.NewGuid().ToString("N")` (per configuration instance) | Override to share STS sessions between containers/instances. |

`AccessKeyId`/`AccessKeySecret`/`Endpoint`/`RegionId`/`RoleArn`/`RoleSessionName`/`Policy` use `GetConfiguration<string>` (throws if unset) — the others use `GetConfigurationOrDefault`.

## `IOssClientFactory` strategy

`DefaultOssClientFactory.GetOssClient(configuration)`:

1. `Check.NotNullOrWhiteSpace` on `AccessKeyId`, `AccessKeySecret` and `Endpoint`.
2. If `UseSecurityTokenService == false` → `return new OssClient(Endpoint, AccessKeyId, AccessKeySecret);`.
3. Otherwise `GetSecurityTokenClient(configuration)`:
   - `Cache.Get(configuration.TemporaryCredentialsCacheKey)` — if hit, decrypt the stored `AccessKeyId`/`AccessKeySecret`/`SecurityToken` via `IStringEncryptionService` and use them.
   - On miss, build `DefaultProfile.GetProfile(RegionId, AccessKeyId, AccessKeySecret)`, create a `DefaultAcsClient(profile)` and send `AssumeRoleRequest { AcceptFormat = JSON, RoleArn, RoleSessionName, DurationSeconds, Policy }`. `Policy` is sent as `null` when the configured string is empty (the role's own permissions then apply unrestricted).
   - Encrypt the three fields of `response.Credentials` via `IStringEncryptionService` and `Cache.Set` with `AbsoluteExpirationRelativeToNow = DurationSeconds - 10` seconds.
   - Build `new OssClient(Endpoint, ak, sk, securityToken)`.

Note that the cached `AliyunTemporaryCredentialsCacheItem` only holds the three encrypted credential strings — it does not persist `response.Credentials.Expiration`. Effective validity is enforced by the cache entry's own TTL (`DurationSeconds - 10s`), matching the AWS provider.

## `AliyunBlobProvider`

| Method | Flow |
| --- | --- |
| `SaveAsync` | Resolve `objectKey = NameCalculator.Calculate(args)` and `bucket = GetContainerName(args)`. If `!OverrideExisting && exists` → `BlobAlreadyExistsException`. If `CreateContainerIfNotExists && !DoesBucketExist` → `PutBucket(bucket)`. Then `oss.PutObject(bucket, objectKey, args.BlobStream)`. |
| `DeleteAsync` | If exists → `DeleteObject(bucket, objectKey)` → true; else false. |
| `ExistsAsync` | `oss.DoesObjectExist(bucket, objectKey)`. |
| `GetOrNullAsync` | If not exists → null. Else `GetObject(bucket, objectKey).Content` is buffered into a `MemoryStream` via `TryCopyToMemoryStreamAsync` (so the SDK response can be disposed). |

`GetContainerName(args)` honours the configured `ContainerName` override (normalised) and otherwise uses `args.ContainerName`.

## `AliyunBlobNamingNormalizer`

OSS bucket name rules: lowercase letters, digits and dashes only; 3-63 chars; must start with a letter or digit. Implementation mirrors the AWS normalizer minus the dot support. Blob (object) names are returned unchanged.

## `DefaultAliyunBlobNameCalculator`

```csharp
return CurrentTenant.Id == null
    ? $"host/{args.BlobName}"
    : $"tenants/{CurrentTenant.Id.Value:D}/{args.BlobName}";
```

## Operational notes

- **Per-call client construction**. `OssClient` is small and cheap to instantiate; it owns its own connection state internally.
- **STS expiration**. Cache TTL is `DurationSeconds - 10` seconds (same pattern as the AWS provider). The 10-second skew leaves room for the round-trip latency between issuing the cached session and Aliyun OSS validating it on the next request.
- **`TemporaryCredentialsCacheKey`** defaults to a per-configuration GUID. Override when you want sub-second reuse across all instances of the same role/policy.
- **Encryption at rest**. `IStringEncryptionService` encrypts the three credential fields in the cache item. Configure `AbpStringEncryptionOptions` consistently.
