`@abp/ng.cms-kit` is the Angular UI for the [CMS Kit module](/modules/cms-kit). It is structured as three secondary entry points — `/admin` (the back-office UI), `/public` (a small placeholder for public-facing fragments), and the umbrella `@abp/ng.cms-kit` (shared utilities, the `ToastUI Editor` and `CodeMirror` editor wrappers). Sources: [`npm/ng-packs/packages/cms-kit/`](https://github.com/abpframework/abp/tree/dev/npm/ng-packs/packages/cms-kit).

## Package layout

```text
packages/cms-kit/
├── package.json           # @abp/ng.cms-kit
├── src/                   # umbrella entry point
│   ├── components/
│   │   ├── toast-ui/toastui-editor.component.ts
│   │   └── code-mirror-editor/code-mirror-editor.component.ts
│   └── utils/
├── admin/                 # @abp/ng.cms-kit/admin
│   └── src/
│       ├── cms-kit-admin.routes.ts
│       ├── components/    # blog-posts/, blogs/, cms-settings/, comments/,
│       │                  # global-resources/, menus/, pages/, tags/
│       ├── defaults/
│       ├── enums/
│       ├── models/
│       ├── resolvers/
│       ├── services/
│       └── tokens/
├── public/                # @abp/ng.cms-kit/public (placeholder routes today)
└── proxy/                 # @abp/ng.cms-kit/proxy
```

| NPM entry point | Purpose |
| --- | --- |
| `@abp/ng.cms-kit` | Editor wrappers + shared utilities |
| `@abp/ng.cms-kit/admin` | Back-office routes + components |
| `@abp/ng.cms-kit/public` | Public-page wrapper (consumer code adds routes here) |
| `@abp/ng.cms-kit/proxy` | Generated services |

Runtime dependencies include `@toast-ui/editor` ^3 and `codemirror` ^6 — the umbrella package wraps them as Angular components.

## Lazy-load configuration

```typescript
// app.routes.ts
{
  path: 'cms-kit',
  loadChildren: () =>
    import('@abp/ng.cms-kit/admin').then(m => m.createRoutes()),
},
{
  path: 'cms',
  loadChildren: () =>
    import('@abp/ng.cms-kit/public').then(m => m.createRoutes()),
},
```

## Admin routes

`packages/cms-kit/admin/src/cms-kit-admin.routes.ts` builds the table below. Every route is `[authGuard, permissionGuard]`-protected, wrapped by `ReplaceableRouteContainerComponent`, and resolved by `cmsKitAdminExtensionsResolver`.

| Path | Required policy | Default component | Replaceable key |
| --- | --- | --- | --- |
| `comments` | `CmsKit.Comments` | `CommentListComponent` | `eCmsKitAdminComponents.CommentList` |
| `comments/:id` | `CmsKit.Comments` | `CommentDetailsComponent` | `eCmsKitAdminComponents.CommentDetails` |
| `tags` | `CmsKit.Tags` | `TagListComponent` | `eCmsKitAdminComponents.Tags` |
| `pages` | `CmsKit.Pages` | `PageListComponent` | `eCmsKitAdminComponents.Pages` |
| `pages/create` | `CmsKit.Pages.Create` | `PageFormComponent` | `eCmsKitAdminComponents.PageForm` |
| `pages/update/:id` | `CmsKit.Pages.Update` | `PageFormComponent` | `eCmsKitAdminComponents.PageForm` |
| `blogs` | `CmsKit.Blogs` | `BlogListComponent` | `eCmsKitAdminComponents.Blogs` |
| `blog-posts` | `CmsKit.BlogPosts` | `BlogPostListComponent` | `eCmsKitAdminComponents.BlogPosts` |
| `blog-posts/create` | `CmsKit.BlogPosts.Create` | `BlogPostFormComponent` | `eCmsKitAdminComponents.BlogPostForm` |
| `blog-posts/update/:id` | `CmsKit.BlogPosts.Update` | `BlogPostFormComponent` | `eCmsKitAdminComponents.BlogPostForm` |
| `menus` | `CmsKit.Menus` | `MenuItemListComponent` | `eCmsKitAdminComponents.Menus` |
| `global-resources` | (settings policy) | `GlobalResourcesComponent` | `eCmsKitAdminComponents.GlobalResources` |

| Component | File | Role |
| --- | --- | --- |
| `ToastUiEditorComponent` | `src/components/toast-ui/toastui-editor.component.ts` | Wraps `@toast-ui/editor` for markdown + rich-text editing |

## Configuration options

`CmsKitAdminConfigOptions` (`admin/src/models/config-options.ts`) supports the standard five contributor maps:

```typescript
{
  entityActionContributors?, toolbarActionContributors?,
  entityPropContributors?,
  createFormPropContributors?, editFormPropContributors?,
}
```

`createRoutes(config)` wires them into the tokens declared in `tokens/extensions.token.ts`:

- `CMS_KIT_ADMIN_ENTITY_ACTION_CONTRIBUTORS`
- `CMS_KIT_ADMIN_TOOLBAR_ACTION_CONTRIBUTORS`
- `CMS_KIT_ADMIN_ENTITY_PROP_CONTRIBUTORS`
- `CMS_KIT_ADMIN_CREATE_FORM_PROP_CONTRIBUTORS`
- `CMS_KIT_ADMIN_EDIT_FORM_PROP_CONTRIBUTORS`

`cmsKitAdminExtensionsResolver` applies them before each child route activates.

## Public entry point

`packages/cms-kit/public/src/lib/cms-kit-public.routes.ts` is currently a scaffold:

```typescript
export function createRoutes(config: CmsKitPublicConfigOptions = {}): Routes {
  return [
    {
      path: '',
      component: RouterOutletComponent,
      providers: provideCmsKitPublicContributors(config),
      children: [ /* Routes will be added here */ ],
    },
  ];
}
```

Public-facing blog/page renderers are normally implemented by the host application, but the wrapper + the proxies under `proxy/src/lib/proxy/volo/cms-kit/` (non-admin variants) are available for that purpose.

## Form services

The Pages and Blog Posts forms are reactive forms with a non-trivial number of fields (slug, title, content, status, tags, layout, blog selection). The package isolates that complexity in dedicated form-state services:

| Service | File | Role |
| --- | --- | --- |
| `PageFormService` | `admin/src/services/page-form.service.ts` | Holds the create/update form group, slug uniqueness validator, content sanitiser, and toggles between markdown / HTML editing modes |
| `BlogPostFormService` | `admin/src/services/blog-post-form.service.ts` | Same for blog posts plus tag autocomplete and blog picker |
| `CommentEntityService` | `admin/src/services/comment-entity.service.ts` | Resolves the entity a comment belongs to so the detail view can deep-link back into the source page or blog post |

Each service is provided at the form-component level (not `root`), so navigating away discards the in-progress state without leaking.

## Settings

The `cms-settings/` folder hosts a settings sub-page that contributes to `SettingTabsService` from `@abp/ng.setting-management/config`. It groups:

- *Comments* (max nesting depth, anonymous comments)
- *Reactions* (allowed reaction types)
- *Global features* (toggle whole sub-features such as Reactions and Comments at deployment level)

The tab is registered through the same provider pattern documented in [`angular/setting-management#settingtabsservice`](/angular/setting-management#settingtabsservice).

## Editor wiring

`PageFormComponent` and `BlogPostFormComponent` both compose the editor:

```html
<abp-toastui-editor
  [(ngModel)]="content"
  [previewStyle]="'vertical'"
  [height]="'500px'"
  [initialEditType]="'markdown'"
  [usageStatistics]="false">
</abp-toastui-editor>
```

`GlobalResourcesComponent` instead uses CodeMirror for the CSS and JS panels:

```html
<abp-codemirror-editor
  [(ngModel)]="css"
  [language]="'css'"
  [theme]="'dark'">
</abp-codemirror-editor>
```

Both wrappers lazy-load their underlying libraries (`@toast-ui/editor` and `codemirror`) via dynamic `import()`, so the initial bundle does not pay the editor cost.
